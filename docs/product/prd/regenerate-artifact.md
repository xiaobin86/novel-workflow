# PRD — 重新生成功能

> **版本**：v1.0  
> **日期**：2026-04-20  
> **状态**：已实现（2026-04-20）  

---

## 1. 背景与目标

当前各阶段执行完成后，如果需要修改生成结果（例如某张图片效果不佳、某段音频有问题），用户只能重新启动整个流程，无法针对单个阶段或单个产物进行局部重新生成。

**目标**：在已完成的步骤中提供两个粒度的重新生成入口：
1. **阶段级重新生成**：一键重新生成该阶段的全部产物
2. **产物级重新生成**：对某一具体产物（单张图片、单个音频、单个视频片段）单独重新生成

---

## 2. 用户故事

> 作为用户，我希望在图片生成阶段完成后，能单独重新生成某一张效果不好的图片，而无需重跑整个图片生成阶段。

> 作为用户，我希望在 TTS 阶段完成后，能整体重新生成所有音频（例如换了配置参数），同时保留其他已完成阶段的产物。

---

## 3. 功能范围

### 3.1 阶段级重新生成

| 触发位置 | 展示条件 | 行为 |
|----------|----------|------|
| 每个步骤卡片底部 | `status === "completed" \|\| status === "stopped"` | 弹出确认框 → 删除该阶段全部产物 → 重新执行该步骤 |

**约束**：
- 仅当该阶段及其后续步骤均未处于 `in_progress` / `paused` 状态时可点击
- 重新生成前删除磁盘上该阶段产物目录（清空 `images/` / `audio/` / `clips/` / `output/`）
- 后续步骤依赖的产物也会被重新生成（不级联，只清本阶段）

### 3.2 产物级重新生成

支持的阶段：

| 步骤 | 产物单元 | 说明 |
|------|----------|------|
| `image` | 单张图片（按 shot_id） | 删除对应 `images/{filename}` 并重新调用服务生成该 shot 的图片 |
| `tts` | 单个 shot 的全部音轨 | 删除对应 `audio/{shotId}_*.mp3` 并重新生成该 shot 的所有音频 |
| `video` | 单个视频片段（按 shot_id） | 删除对应 `clips/{filename}` 并重新生成该 shot 的视频 |
| `storyboard` / `assembly` | 不支持产物级重新生成 | 分镜和合并只能整体重新生成 |

**触发方式**：每个产物卡片右上角悬浮「🔄」图标，点击后确认 → 执行产物级重新生成。

---

## 4. API 设计

### 4.1 阶段级重新生成

#### `POST /api/pipeline/[id]/[step]/reset`

删除该步骤的磁盘产物并将步骤状态重置为 `pending`。前端随后调用现有的 `start` 接口启动新任务。

**处理逻辑**：
1. 读取 state.json，检查该步骤是否处于活跃状态（`in_progress`/`paused`）
2. 如活跃则返回 `409 Conflict`
3. 删除该步骤对应产物目录（见下表）
4. 将该步骤状态重置：`{ status: "pending", job_id: null, result: null }`
5. 返回 `200 OK`

**各步骤产物目录**：

| 步骤 | 删除路径 |
|------|----------|
| `storyboard` | `storyboard.json` |
| `image` | `images/` 目录全部文件 |
| `tts` | `audio/` 目录全部文件 |
| `video` | `clips/` 目录全部文件 |
| `assembly` | `output/` 目录全部文件 |

**错误响应**：

| 状态码 | 原因 |
|--------|------|
| 404 | 项目不存在 |
| 409 | 步骤正在执行中 |
| 500 | 文件系统错误 |

---

### 4.2 产物级重新生成

#### `POST /api/pipeline/[id]/[step]/regenerate-item`

**请求 Body**：
```json
{
  "shot_id": "shot_001"
}
```

**处理逻辑**：
1. 读取 storyboard.json，验证 `shot_id` 存在
2. 删除该 shot 对应的产物文件（见下表）
3. 调用该步骤的后端服务，仅生成指定 shot（在 `config` 中传入 `shot_ids: [shot_id]`）
4. 更新 state.json：步骤状态置为 `in_progress`，记录新 `job_id`
5. 返回 `{ job_id }` 和 `202 Accepted`

**各步骤产物文件**：

| 步骤 | 删除文件 |
|------|----------|
| `image` | `images/{shotId}*.{png,jpg,webp}` |
| `tts` | `audio/{shotId}_*.mp3` |
| `video` | `clips/{shotId}*.mp4` |

**错误响应**：

| 状态码 | 原因 |
|--------|------|
| 400 | shot_id 不存在 |
| 404 | 项目不存在 |
| 409 | 步骤正在执行中 |
| 500 | 文件系统或服务错误 |

## 5. UI 设计

### 5.1 阶段级重新生成按钮

在步骤卡片底部操作区，当 `status === "completed" || status === "stopped"` 时，现有的操作区显示：

```
[ 重新生成全部 ↺ ]
```

- 样式：`variant="outline"` + 小字，与下一步"确认并继续"按钮并排
- 点击后弹出确认框（复用 Dialog 组件）：

```
┌─────────────────────────────────────────┐
│  重新生成「图片生成」                     │
│                                         │
│  此操作将删除该阶段所有已生成产物，       │
│  且不可恢复。确认后将立即重新执行。        │
│                                         │
│          [ 取消 ]  [ 确认重新生成 ]       │
└─────────────────────────────────────────┘
```

### 5.2 产物级重新生成图标

在每个产物卡片（图片/音频/视频）右上角，hover 时显示 🔄 图标按钮：

- 图片卡片：右上角与 `shot_id` 标签相对的位置
- 音频行：右侧
- 视频卡片：右上角

点击后无需确认框，直接触发（操作轻量）。执行时对应产物位置显示 loading 占位。

---

## 6. 非功能需求

- 阶段级重置后台立即返回（<200ms），用户感知流畅
- 产物级重新生成期间其他已完成产物正常展示，不影响整体状态展示
- 重新生成期间步骤状态变为 `in_progress`，进度条和实时预览正常工作

---

## 7. 不在范围内

- 级联重新生成（重新生成图片时自动重新生成依赖它的视频和合并）— v1.0 不做
- 批量选择多个产物重新生成 — v1.0 不做
- 自定义生成参数（如切换图片尺寸）— 复用现有配置
