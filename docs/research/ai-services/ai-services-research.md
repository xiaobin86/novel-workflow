# 付费 AI 服务调研报告：分镜生成、图片生成、视频生成

> 调研日期：2026年4月
> 调研目的：为 Novel Workflow 项目寻找可替代本地 GPU 方案的付费云端 AI 服务
> 当前方案：Kimi API (分镜) + 本地 FLUX.1-dev 4-bit (图片) + 本地 Wan2.1-T2V-1.3B (视频)
> 硬件瓶颈：RTX 5070 Ti 12GB，图片~15分钟/10张，视频~50分钟/10段

---

## 1. 执行摘要

| 类别 | 最佳选项（预算充足） | 最佳选项（性价比） | 核心改进 |
|------|---------------------|-------------------|---------|
| **分镜生成** | Claude Sonnet 4.6 / GPT-4o | Kimi K2.5（当前方案已优秀） | 结构化输出、角色分析、镜头语言 |
| **图片生成** | fal.ai FLUX.2 [pro] + LoRA | fal.ai FLUX.2 [klein] / Together AI | 云端 5-10秒生成，多参考图一致性 |
| **视频生成** | Runway Gen-4 / 可灵 3.0 | Wan2.1 云端 API (WaveSpeed/Replicate) | 速度提升 10-30倍，1080p/4K 可选 |
| **角色一致性** | FLUX.2 + Multi-Reference / Midjourney --cref | LoRA 训练 (VicSee/Scenario) | 批量生成时保持角色外观一致 |

**一句话总结**：云端方案可将总生产时间从数小时压缩至分钟级，成本约为每 10-shot 场景 $2-8（图片+视频），分镜成本可忽略。

### 2.3 分镜生成成本估算（10-shot 场景）

以 10 个镜头、每个镜头 500 token 输入 / 200 token 输出计算：

| 服务 | 单次成本 | 10-shot 成本 | 速度 |
|------|---------|-------------|------|
| Kimi K2.5 | ~$0.0009 | ~$0.009 | <3秒 |
| Claude Sonnet 4.6 | ~$0.0045 | ~$0.045 | <5秒 |
| GPT-4o | ~$0.003 | ~$0.03 | <3秒 |
| DeepSeek V3.2 | ~$0.0002 | ~$0.002 | <5秒 |

**分析**：
- **Kimi K2.5** 在中文小说理解和成本上具有绝对优势，当前方案已非常合理。
- 如需提升分镜质量，可引入 **Claude Sonnet 4.6** 或 **GPT-4o** 作为备选 Provider，通过提示词工程强化镜头语言和角色一致性控制。
- **Drawstory** 等专业工具适合没有技术团队的创作者，但对于 Novel Workflow 的自动化流水线，直接调用 LLM API 更灵活。

## 5. 综合建议

### 5.1 如果预算充足，最佳组合是什么？

**「专业级流水线」组合**

| 环节 | 服务 | 月估算成本（100 scenes） |
|------|------|------------------------|
| 分镜 | Claude Sonnet 4.6 | ~$5 |
| 图片 | fal.ai FLUX.2 [pro] + Multi-Reference | ~$30-50 |
| 视频 | Runway Gen-4 I2V + 可灵 3.0 备选 | ~$200-300 |
| **总计** | | **~$235-355/月** |

**优势**：
- 分镜质量最高，结构化输出最稳定
- 图片角色一致性通过 FLUX.2 Multi-Reference 实现，无需训练 LoRA
- 视频质量行业顶尖，Runway 的 Motion Brush 和 References 功能提供精细控制
- 总生产时间：10-shot 场景从数小时压缩至 **5-10分钟**

### 5.2 如果追求性价比，最佳组合是什么？

**「高效流水线」组合**

| 环节 | 服务 | 月估算成本（100 scenes） |
|------|------|------------------------|
| 分镜 | Kimi K2.5（当前方案） | ~$1-2 |
| 图片 | fal.ai FLUX.2 [klein] + Multi-Reference | ~$15-25 |
| 视频 | WaveSpeed AI / Replicate Wan 2.1 I2V | ~$20-40 |
| **总计** | | **~$36-67/月** |

**优势**：
- 成本仅为本地 GPU 电费的几倍，但速度提升 **30-60倍**
- FLUX.2 [klein] 4B 模型速度最快（<10秒），质量足以满足动漫风格
- Wan 2.1 云端 API 与本地模型一致，迁移成本最低
- 总生产时间：10-shot 场景约 **3-5分钟**

### 5.3 哪些服务适合作为 Provider 抽象层的云端实现？

适合集成到 FastAPI Provider 架构的服务：

| 服务 | 集成难度 | 推荐理由 |
|------|---------|---------|
| **Kimi K2.5** | 低 | OpenAI 兼容 API，当前已使用 |
| **fal.ai** | 低 | REST API，Python/JS SDK，Webhook 支持 |
| **Together AI** | 低 | OpenAI 兼容 API，统一 SDK |
| **Replicate** | 低 | REST API，丰富的模型库 |
| **WaveSpeed AI** | 中 | 专业视频 API，文档完善 |
| **可灵 API** | 中 | 国内访问快，中文文档，企业级支持 |
| **VicSee** | 中 | Character LoRA API，适合角色一致性 |

**不建议作为 Provider 的服务**：
- Midjourney：无官方 API，第三方 API 不稳定且可能违反 ToS
- Runway：API 较封闭，主要面向 SaaS 平台用户
- 海艺 AI：无开放 API

## 6. 技术可行性分析

### 6.1 无缝集成（低工作量）

| 服务 | 集成方式 | 预计工作量 | 关键代码示例 |
|------|---------|-----------|-------------|
| **Kimi K2.5** | OpenAI 兼容 API | 已集成 | `openai.AsyncClient(base_url="...")` |
| **fal.ai** | Python SDK / REST | 1-2天 | `fal_client.subscribe("fal-ai/flux/dev", args)` |
| **Together AI** | OpenAI 兼容 API | 1天 | `together.images.generate(model="black-forest-labs/FLUX.2-pro", ...)` |
| **Replicate** | Python SDK | 1-2天 | `replicate.run("wavespeedai/wan-2.1-i2v-480p", input={...})` |

### 6.2 需要额外工作量的集成

| 服务 | 额外工作 | 预计工作量 | 说明 |
|------|---------|-----------|------|
| **可灵 API** | 签名认证，国内网络优化 | 2-3天 | 需要处理国内 API 的认证流程 |
| **LoRA 训练（VicSee/Scenario）** | 训练工作流，角色管理 | 3-5天 | 需要上传角色图，等待训练，管理 character_id |
| **FLUX.2 Multi-Reference** | 参考图上传/URL 管理 | 1-2天 | 需要先将角色图上传到可公开访问的 URL |
| **Midjourney（第三方）** | 轮询机制，Rate Limit 处理 | 3-5天 | 无官方 API，可靠性风险 |

### 6.4 迁移建议

   - 收益：速度从 15分钟 → 50秒，一致性大幅提升

   - 改动：修改 Provider 配置，添加 `image` 参数
   - 收益：速度从 50分钟 → 10分钟，分辨率可达 720p

3. **角色一致性**：
   - 方案 A（快速）：使用 FLUX.2 Multi-Reference，每次生成时传入角色参考图 URL
