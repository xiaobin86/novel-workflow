## 2. UI 设计亮点分析

### 2.1 值得参考的设计模式

#### A. 分步骤引导（Wizard）模式

**最佳实践来源：漫剧助手、AI Comic Builder、剪映图文成片**

这些工具都采用了清晰的分步引导设计：

- **漫剧助手**：小说→剧本→设定→分镜→素材生成，五步流程，每步有明确的入口和状态标识
- **AI Comic Builder**：剧本输入→角色提取→智能分镜→首尾帧→视频生成→视频合成，每个阶段可单独触发或批量生成
- **剪映图文成片**：输入文案→生成视频→预览编辑→导出，极简三步流程

**关键设计模式**：
- 顶部或侧边进度条显示当前所处阶段
- 每个步骤有明确的"完成"状态和"可编辑"状态
- 支持在步骤间跳转（非线性导航）
- 步骤之间有"下一步/上一步"的明确导航按钮
- 长流程支持"保存草稿"功能

#### B. 节点式/看板式状态展示

**最佳实践来源：ComfyUI、AI Comic Builder**

- **ComfyUI**：每个节点有明确的执行状态（等待中/运行中/完成/错误），节点边框颜色和图标状态实时变化
- **AI Comic Builder**：分镜页提供**列表视图**和**看板视图**，看板按生成进度自动分列（待生成/生成中/已完成/失败）

**关键设计模式**：
- 用颜色编码状态：灰色（未开始）、蓝色（进行中）、绿色（完成）、红色（失败）
- 节点/卡片上显示实时进度百分比或旋转加载动画
- 失败状态提供"重试"按钮和错误详情入口
- 支持批量操作（批量生成、批量重试）

#### C. 素材预览与网格管理

**最佳实践来源：Premiere Pro、DaVinci Resolve、Runway**

- **Premiere Pro 项目面板**：支持列表视图、图标视图、自由视图三种模式；图标视图支持悬停 scrub 预览
- **DaVinci Resolve 媒体池**：支持元数据视图、缩略图视图、列表视图；可手动拖拽排序；支持智能分类(Smart Bins)
- **Runway**：生成结果以卡片网格展示，每个卡片有预览图、时长、分辨率信息

**关键设计模式**：
- 缩略图网格 + 悬停播放预览
- 列表视图显示元数据（名称、时长、状态、创建时间）
- 支持多选、拖拽排序、批量删除
- 素材右键菜单提供快捷操作（复制、删除、重命名、查看详情）

#### D. 角色/主体一致性管理

**最佳实践来源：漫剧助手、Anime AI Studio、AI Comic Builder**

- **漫剧助手**："多状态"与"主体库"功能，深度角色设定体系，从设定到成片全程一致性保障
- **Anime AI Studio**：基于LoRA模型，在数千张图片中保持角色形象一致，支持角色四视图
- **AI Comic Builder**：角色提取后生成"角色三视图/四视图"作为后续生成的参考

**关键设计模式**：
- 独立的角色管理面板，显示角色头像、名称、描述
- 角色详情页包含参考图（多角度的角色设定图）
- 角色与分镜/场景关联，在生成分镜时自动应用对应角色参考
- 支持角色跨项目复用

#### E. 无限画布/自由工作区

**最佳实践来源：Anime AI Studio、ComfyUI**

- **Anime AI Studio**："无限画布工作区"，非线性制作空间，故事可以向四面八方生长，不受页面限制
- **ComfyUI**：无限画布节点编辑器，支持缩放、平移、框选、对齐

**关键设计模式**：
- 支持缩放（鼠标滚轮）和平移（拖拽空白处）
- 内容可以自由放置在画布任意位置
- 适合分镜的"故事板"式布局

### 2.2 用户体验好的交互方式

#### A. 生成进度实时反馈

**优秀案例：Pika Labs、Runway、AI Comic Builder**

- Pika：生成时显示"Generating..."动画和预计剩余时间
- Runway：生成队列系统，显示当前排队位置和处理进度
- AI Comic Builder："批量生成，实时进度追踪"，每个任务有独立进度条

**推荐模式**：
- 对于长耗时任务（视频生成），显示**实时进度条** + **预计剩余时间**
- 支持**后台运行**，用户可以继续浏览其他页面
- 完成后通过**视觉提示**（toast通知、徽章数字）提醒用户
- 提供**取消任务**按钮

#### B. 一键/批量操作

**优秀案例：漫剧助手、剪映**

- 漫剧助手："一键批量直出"涵盖图片/视频/音频等多模态素材
- 剪映图文成片：一键从文字生成完整视频

**推荐模式**：
- 提供"一键生成全部"的快捷按钮
- 同时支持"单步精细控制"，满足不同用户需求
- 批量操作时显示总体进度和每个子任务的状态

#### C. 实时预览与快速迭代

**优秀案例：Runway、Pika**

- Runway：生成5秒视频后可立即预览，不满意可调整参数重新生成
- Pika：支持3次迭代法（探索→优化→定型）

**推荐模式**：
- 每个生成环节都有独立的**预览区域**
- 支持**参数微调后快速重生成**
- 保留历史版本，方便对比不同结果

### 2.3 适合 Pipeline 场景的可视化方式

#### A. 管道流程图（Pipeline Flow）

**最佳实践来源：AI Comic Builder、漫剧助手**

AI Comic Builder 的流水线可视化：
```
剧本输入 → 剧本解析 → 角色提取 → 角色四视图
                                      ↓
                                   智能分镜
                                      ↓
                                首尾帧/场景参考帧
                                      ↓
                                视频提示词 → 视频生成
                                      ↓
                                视频合成 + 字幕
```

**推荐模式**：
- 用水平或垂直流程图展示 Pipeline 各阶段
- 每个阶段显示状态图标和统计数字（如"12/15 完成"）
- 点击每个阶段可展开查看详情
- 用连线表示数据流向

#### B. 分镜时间线（Storyboard Timeline）

**最佳实践来源：Premiere Pro、DaVinci Resolve、AI Comic Builder**

- 分镜按时间顺序排列
- 每个分镜显示缩略图、时长、对白/旁白文字
- 支持拖拽调整顺序
- 时间轴标尺显示总时长

## 3. 可借鉴的具体设计元素

### 3.1 步骤引导/Wizard 设计

| 设计元素 | 来源工具 | 具体实现 | 适用场景 |
|---------|---------|---------|---------|
| **顶部进度条** | 漫剧助手、剪映 | 水平步骤条，每步有圆形编号和名称 | Pipeline 主流程导航 |
| **侧边步骤导航** | LangSmith Studio | 左侧步骤列表，可点击跳转，带完成标记 | 多步骤配置页面 |
| **步骤卡片** | AI Comic Builder | 每个步骤为一张卡片，显示状态和操作按钮 | 仪表盘式首页 |
| **流程图高亮** | ComfyUI | 运行中的节点高亮显示，已完成节点灰色处理 | 实时执行状态展示 |

**对我们项目的建议**：
- 在首页/仪表盘使用**水平步骤进度条**，清晰展示"文本→分镜→图片→音频→视频→成片"六步流程
- 每个步骤显示**当前状态**（未开始/进行中/已完成）和**快捷操作按钮**
- 点击已完成的步骤可**回退修改**，点击进行中的步骤可**查看详情**
- 用**微动画**（如脉冲光效）吸引用户注意当前活跃步骤

### 3.2 进度展示（长耗时任务）

| 设计元素 | 来源工具 | 具体实现 | 适用场景 |
|---------|---------|---------|---------|
| **进度条+百分比** | Runway、Pika | 水平进度条显示百分比，下方显示"预计剩余X分钟" | 视频/图片生成 |
| **队列位置** | Runway | "队列中第3位，预计等待2分钟" | 资源排队场景 |
| **多任务并行进度** | AI Comic Builder | 多个卡片同时显示各自进度 | 批量生成场景 |
| **后台运行提示** | 剪映 | 底部固定栏显示"正在生成视频..."，可点击展开详情 | 允许用户继续其他操作 |
| **完成通知** | DaVinci Resolve | 右上角Toast通知，带"查看"和"关闭"按钮 | 任务完成提醒 |

**对我们项目的建议**：
- **全局任务栏**：在页面顶部或侧边固定一个"任务队列"面板，显示所有正在运行和待运行的任务
- **分镜卡片进度**：每个分镜卡片上显示小进度条（当该分镜正在生成图片/视频时）
- **预估时间**：基于历史数据，显示"预计剩余X分钟"
- **通知中心**：任务完成后发送浏览器通知 + 应用内通知
- **失败处理**：失败任务显示红色状态，提供"重试"和"查看日志"按钮

### 3.3 素材预览（图片网格、音频播放、视频播放）

## 4. 不推荐的设计

### 4.1 ComfyUI 的节点式工作流（对普通用户）

**问题**：
- 学习曲线极陡，需要理解Stable Diffusion的技术概念（Latent、VAE、KSampler等）
- 界面信息密度过高，50+节点的复杂图表让新手望而却步
- 缺少引导式流程，用户不知道从何开始

**适用场景**：适合技术用户和AI研究者，**不适合**面向普通创作者的小说漫剧工具。

**替代方案**：在底层可以借鉴ComfyUI的节点执行引擎，但在UI层应该包装为**向导式界面**。

### 4.2 Pika 的纯生成循环（缺乏项目管理）

**问题**：
- Pika的核心是"prompt → generate → refine"的单次生成循环
- 缺少项目概念，生成的片段分散在会话历史中
- 不支持将多个片段组织成一个完整的故事/剧集

**适用场景**：适合快速生成单个短视频片段，**不适合**需要多集/多章节管理的长篇小说漫剧制作。

**替代方案**：需要建立**项目-章节-分镜**的三级结构来组织内容。

### 4.3 过度复杂的设置面板

**问题**：
- 部分AI工具（如ComfyUI、Stable Diffusion WebUI）提供大量技术参数（CFG Scale、Steps、Sampler等）
- 普通用户难以理解这些参数的含义
- 过多的选择反而导致选择困难

**替代方案**：
- 提供**预设模式**（快速/标准/高质量）
- 高级参数默认隐藏，通过"高级设置"展开
- 参数调整提供实时效果预览和中文说明

### 4.4 缺乏角色一致性的单一生成模式

**问题**：
- 早期AI漫画/视频工具（如基础的Stable Diffusion）每次生成都独立随机
- 角色在不同分镜中外观不一致，破坏叙事连贯性
- 需要用户手动使用LoRA/ControlNet等技术手段解决

**替代方案**：
- 内置**角色提取和管理系统**
- 自动生成角色参考图（三视图/四视图）
- 在生成分镜时自动注入角色一致性控制

## 5. 对我们项目的优化建议

### 5.1 核心设计原则

基于以上调研，建议 Novel Workflow 的 Web UI 遵循以下设计原则：

1. **渐进式披露**：简单任务一键完成，复杂任务层层展开
2. **可视化优先**：用缩略图、进度条、状态颜色替代纯文本信息
3. **实时反馈**：每个操作都有即时响应，长任务有进度追踪
4. **容错设计**：失败的任务可重试，重要操作可撤销
5. **一致性体验**：相同类型的操作在不同页面有相同交互方式

### 5.2 页面结构建议

```
Novel Workflow Web UI
```

### 5.3 关键界面详细建议

#### A. 仪表盘首页

借鉴：**剪映首页 + DaVinci Resolve Project Manager**

- **"+ 新建项目"** 大按钮放在视觉中心
- **"最近项目"** 横向滚动卡片，显示项目封面、名称、进度
- **"从模板开始"** 区域，提供常见题材模板（古风、都市、悬疑等）
- **"使用指南"** 入口，首次使用时突出显示

#### B. Pipeline 进度导航

借鉴：**漫剧助手 + AI Comic Builder**

- 固定在项目编辑器顶部
- 6个步骤：📖剧本 → 🎬分镜 → 🖼️图片 → 🔊音频 → 🎞️视频 → ✅成片
- 每个步骤显示：**状态图标** + **完成数量/总数**（如"分镜 12/15"）
- **当前步骤**高亮显示，带脉冲动画
- 点击已完成步骤可回退到该步骤编辑
- 步骤之间用箭头连接，清晰展示数据流向

#### C. 分镜网格页

借鉴：**AI Comic Builder + Premiere Pro 项目面板**

- **三种视图模式**：
  - **网格视图**：等宽卡片网格，适合浏览全局
  - **看板视图**：按状态分列（待生成/生成中/已完成/失败），适合管理工作流
  - **时间线视图**：横向时间轴，适合调整节奏和顺序
- **每个分镜卡片显示**：
  - 缩略图（图片/视频第一帧）
  - 分镜编号（如"SC-001"）
  - 时长（如"5s"）
  - 状态标签（颜色编码）
  - 音频图标（如已生成配音）
- **批量操作栏**：全选/批量生成/批量删除/导出
- **拖拽排序**：支持拖拽调整分镜顺序

#### D. 角色管理面板

借鉴：**漫剧助手主体库 + Anime AI Studio**

- **角色列表**：头像 + 名称 + 角色类型（主角/配角）
- **角色详情页**：
  - 左侧：角色参考图（三视图/四视图网格）
  - 右侧：角色信息（名称、描述、特征标签）
  - 底部：关联分镜列表
- **添加角色**：支持手动创建或从剧本自动提取
- **角色一致性开关**：开启后在生成分镜时自动使用该角色的参考图

#### E. 素材预览与播放器

借鉴：**Runway + DaVinci Resolve**

- **图片预览**：
  - 点击后弹出 Lightbox 查看器
  - 支持缩放、对比（原图 vs 生成图）
  - 显示生成参数（使用的模型、提示词等）
- **视频预览**：
  - 内联播放器：鼠标悬停显示播放按钮，点击在卡片内播放
  - 全屏播放器：点击展开全屏，带播放控制条（播放/暂停/进度/音量）
  - 字幕开关：可切换显示/隐藏字幕
- **音频预览**：
  - 显示音频波形图
  - 点击播放/暂停
  - 显示音频时长和语速信息

#### F. 任务队列与通知

借鉴：**Runway 队列 + 剪映后台生成**

- **全局任务浮窗**：
  - 固定在页面右下角
  - 显示当前运行任务数和排队任务数
  - 展开后显示任务列表（名称、进度、状态）
  - 支持取消任务
- **通知中心**：
  - 页面右上角铃铛图标
  - 任务完成/失败时弹出 Toast 通知
  - 通知列表显示历史消息

### 5.4 交互细节建议

#### A. 生成按钮设计

- **主生成按钮**：大号按钮，使用品牌色，文案清晰（"生成分镜"、"生成视频"）
- **批量生成**：下拉菜单或拆分按钮（"生成全部" / "生成选中项"）
- **重新生成**：已完成的项提供"重新生成"按钮，hover 时显示

#### B. 空状态设计

- **剧本页面为空**：显示"上传小说文件或粘贴文本开始创作"，配插图
- **分镜页面为空**：显示"剧本解析后将自动生成分镜"，配 loading 动画
- **素材库为空**：显示"开始生成后素材将出现在这里"

#### C. 加载状态设计

- **骨架屏**：页面首次加载时显示骨架屏而非空白
- **局部加载**：卡片内生成时显示局部 loading（如缩略图区域旋转动画）
- **进度提示**：长任务显示"这可能需要几分钟，您可以先浏览其他页面"

#### D. 错误处理设计

- **友好错误提示**：不显示技术错误代码，显示"生成失败，请重试"或"网络异常，请检查连接"
- **重试机制**：失败任务自动显示"重试"按钮，支持批量重试
- **降级方案**：图片生成失败时显示占位图，不阻塞后续流程

### 5.5 技术实现建议

- **前端框架**：React + TypeScript（与现有项目一致）
- **状态管理**：Zustand（AI Comic Builder 使用，适合中等复杂度状态）
- **UI 组件库**：
  - 基础组件：shadcn/ui 或 Ant Design
  - 无限画布：React Flow 或自研 Canvas（如需节点编辑器）
  - 看板：@hello-pangea/dnd 或 react-beautiful-dnd
- **实时通信**：WebSocket 用于任务进度推送
- **视频播放**：Video.js 或原生 HTML5 Video
- **图片预览**：react-image-lightbox 或自研 Lightbox
