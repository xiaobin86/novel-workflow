# v1.0 Core MVP — 产品需求文档

## 1. 产品概述

**产品名称**：Novel Workflow  
**定位**：个人使用的小说漫剧生成工具，将小说文本自动化生产为带有配音和字幕的动漫风格短视频。

### 两个核心目标

| 目标 | 说明 |
|------|------|
| **后端模型服务** | 5 个 Docker 微服务，封装 AI 大模型能力，对外暴露标准 HTTP 接口 |
| **Web UI 操作界面** | 可视化流程管理，分步引导用户完成小说到视频的完整链路 |

---

## 2. 目标用户

个人创作者（单用户），有本地 GPU 资源，熟悉基本操作，希望把喜欢的小说内容制作成动漫漫剧风格的短视频。

---

## 3. 产品目标（v1.0）

1. 打通「小说文本 → 分镜 → 图片 → 配音 → 视频片段 → 最终成片」完整链路
2. 每个步骤的产物可在界面预览，用户确认后才继续下一步
3. 支持多部小说项目管理，随时暂停、恢复某个项目的进度
4. 支持从任意步骤切入（只要该步骤的输入满足格式要求）

---

## 4. 功能范围

### 4.1 后端模型服务（5 个 Docker 服务）

每个服务有统一的设计原则：
- 标准 HTTP 接口（FastAPI）
- SSE（Server-Sent Events）实时推送处理进度
- Provider 抽象层，当前实现本地模型，预留云端扩展接口
- 读写共享的项目文件目录

| 服务 | 端口 | 核心能力 | 模型/工具 |
|------|------|---------|----------|
| storyboard-service | 8001 | 小说文本 → 分镜 JSON | Kimi API (kimi-k2.5) |
| image-service | 8002 | 分镜 → 图片 PNG | FLUX.1-dev（本地 GPU） |
| tts-service | 8003 | 分镜文本 → 旁白/对话 WAV | edge-tts（Azure 免费） |
| video-service | 8004 | 分镜 → 视频片段 MP4 | Wan2.1-T2V-1.3B（本地 GPU） |
| assembly-service | 8005 | 全部素材 → 最终 MP4 + SRT | FFmpeg |

#### 服务执行约束
- image-service 和 video-service 共用同一张 GPU（12GB VRAM），**必须串行**，不得并行
- tts-service 仅用 CPU，可与 image-service **并行**运行（实现细节，UI 侧按顺序展示即可）
- video-service 执行前必须先完成 tts-service（视频时长依赖 TTS 音频实际时长计算）

#### v1.0 固定参数（不暴露配置）
- TTS 声线：默认（`YunxiNeural` 旁白，`XiaoxiaoNeural` 对话）
- 图片风格：默认 anime chinese manhua style
- 生成分辨率：图片 768×768，视频 832×480
- BGM：不在 v1.0 范围内

### 4.2 Web UI（Next.js）

#### 项目管理
- 项目列表页：展示所有小说项目，包含名称、当前进度、创建时间
- 支持新建项目、继续已有项目、删除项目
- 每个项目独立保存进度状态（`projects/{id}/state.json`）

#### 流程向导（Pipeline Wizard）
5 个步骤按顺序展示，每步包含：
- 当前状态指示（待开始 / 进行中 / 已完成 / 失败）
- 执行按钮（开始/重新生成）
- 进度展示（实时，通过 SSE 推送）
- 产物预览（完成后展示，见下方各步骤说明）
- 确认继续按钮（确认后才解锁下一步）

**全局自动模式开关**：开启后，每步完成自动进入下一步，无需手动确认（适合无人值守运行）。

#### 各步骤产物预览

| 步骤 | 预览形式 |
|------|---------|
| 分镜生成 | 分镜列表，每条显示 shot_id、场景描述、旁白文本、对话文本 |
| 图片生成 | 图片网格，每张对应一个镜头，可点击查看大图 |
| TTS 生成 | 每个镜头的旁白/对话音频可播放 |
| 视频生成 | 每个镜头的视频片段可播放 |
| 素材拼装 | 最终 MP4 视频播放器 + 字幕文件下载 |

#### 断点续传
- 每步完成的产物文件持久化到项目目录
- 服务重启或失败后，已完成的镜头不重新生成
- 失败的单个镜头可单独重试

#### 跳入模式（从任意步骤开始）
用户可跳过前序步骤，直接上传符合格式的中间产物从某步开始：
- 上传分镜 JSON → 从图片生成开始
- 上传分镜 JSON + 图片 → 从 TTS 生成开始
- 上传分镜 JSON + 图片 + 音频 → 从视频生成开始
- 上传所有素材 → 直接拼装

## 5. 核心用户流程

### 流程 A：完整新建流程
```
新建项目（填写标题）
  → 上传/粘贴小说文本
  → [生成分镜] → 预览分镜列表 → 确认
  → [生成图片] → 预览图片网格 → 确认
  → [生成TTS] → 试听音频 → 确认
  → [生成视频片段] → 预览视频 → 确认
  → [素材拼装] → 播放最终成片 → 下载
```

### 流程 B：自动模式（无需逐步确认）
```
新建项目 → 上传文本 → 开启自动模式 → 点击「开始全流程」
  → 后台依次执行所有步骤 → 完成后通知
```

### 流程 C：继续已有项目
```
项目列表 → 选择未完成项目
  → 进入向导，定位到最后完成的步骤
  → 从该步骤继续
```

### 流程 D：从中间步骤切入
```
新建项目 → 选择「从分镜开始」
  → 上传 storyboard.json
  → 直接进入图片生成步骤
```

---

## 6. 非功能需求

| 项目 | 要求 |
|------|------|
| 进度可见性 | 长耗时步骤（图片/视频）实时展示每个镜头的处理进度 |
| 错误提示 | 单镜头失败时明确提示，可单独重试，不影响其他镜头 |
| 并发 | 单用户，无需多任务并发，同一时间只运行一个步骤 |
| 数据持久化 | 项目数据和产物文件保存在本地，关闭浏览器不丢失 |

---

## 7. v1.0 不在范围内（Out of Scope）

- BGM 生成和配置
- TTS 声线自定义
- 图片/视频风格参数调整
- 多用户/权限管理
- 云端 AI 服务（Provider 接口预留，但 v1.0 只实现本地）
- 分镜内容的界面内编辑（只可预览，如需修改编辑 JSON 文件）
- 视频导出格式选择
